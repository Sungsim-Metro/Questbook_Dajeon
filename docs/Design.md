# 필수 요구사항

한국관광공사의 OpenAPI를 필수 사용
특히 OpenAPI의 데이터를 받아와서 서버에 저장 또는 처리 후 저장하여 사용하는 방식이 아닌

# 기능

1. 유저의 취향에 따른 주변 관광지 추천
2. 관광지별 퀘스트 생성 및 제공
3. 퀘스트 클리어에 따른 뱃지 및 레벨 시스템
4. 이전에 성공한 퀘스트 내역 기반의 모험가의 수첩

# 관광지 추천 방식

유저의 위치와 OpenAPI를 통한 주변 관광지 정보 수집 -> 유저의 이전 퀘스트 내역과 입력한 선호도에 따른 관광지 필터링 -> 적절한 관광지 추천

# 퀘스트 생성 방식

1. 사전에 제작해둔 템플릿 중 관광지의 카테고리에 맞는 템플릿들 선택(음식점, 산책로, 박물관 등) -> 템플릿에 관광지 정보를 적절하게 삽입 -> 생성된 퀘스트 점검
2. 생성형 AI를 통한 퀘스트 생성 -> 퀘스트의 적절성 판단 및 점수 부여
3. 이전에 다른 사용자를 위하여 생성된 퀘스트를 저장하여 재활용

# 뱃지 및 레벨 시스템

## 뱃지

퀘스트 카테고리별 뱃지를 사전 생성
퀘스트 클리어시에 획득하는 점수를 통해 해당 카테고리의 뱃지의 요구 점수 채우고 충족 시 뱃지 부여
뱃지별로 단계 구성되어있음 (ex.  음식탐방 Lv.1 -> Lv.2 또는 음식 소비자 -> 음식 탐험가 -> 미식 탐험가 등)
단계는 획득과 마찬가지로 퀘스트 수행 후 획득하는 점수로 업그레이드

## 레벨

퀘스트 수행하면 획득하는 점수로 업그레이드
